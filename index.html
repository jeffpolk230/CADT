I heard you liked ADTs, so I put some in your C.

CADT is a preprocessor for generating Haskell-like ADTs in C. For instance, this example ADT definition

data Expr = Plus { l : int, r : int }
          | Inc { o : int }

licenses the following C code:

int evaluate(struct Expr e) {
  switch (e.tag) {
    case Plus: return e.Plus.l + e.Plus.r;
    case Inc:  return e.Inc.o + 1;
  };

  return -1;
}

void test() {
  struct Expr e1 = Expr.Plus(1,3);
  struct Expr e2 = Expr.Inc(5);
  assert(evaluate(e1) == 4);
  assert(evaluate(e2) == 6);
}

Recursive types?

Recursive types are not currently possible with CADT; when I get around to it, they'll have to be implemented with pointers I suppose.

Pattern matching syntax

Patience, I'm working on it. I'm hoping for destructuring by constructor name (as in Haskell). This will eliminate the awkwardness of having to access the union members.

How this works

Basically, the above ADT definition compiles down to the following C:

struct Expr {
  enum {Plus,Inc} tag;
  union {
    struct {
      int l;
      int r;
    } Plus;
    struct {
      int o;
    } Inc;
  };
};
inline struct Expr Expr__Plus(int l,int r) {
  struct Expr x = {Plus};
  x.Plus.l = l;
  x.Plus.r = r;
  return x;
}
inline struct Expr Expr__Inc(int o) {
  struct Expr x = {Inc};
  x.Inc.o = o;
  return x;
}
const struct {
  struct Expr (*Plus)(int l,int r);
  struct Expr (*Inc)(int o);
} Expr = {
  .Plus = &Expr__Plus,
  .Inc = &Expr__Inc
};
